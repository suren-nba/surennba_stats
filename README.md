# NBA 数据采集库 (NBA Data Spiders)

这是我的 NBA 数据抓取开源项目。本仓库主要用于托管 [surennba.com](https://www.surennba.com) 网站的核心数据源抓取脚本。

该内容完全基于 Python 开发，旨在降低获取和清洗 NBA 高阶数据的门槛。所有抓取逻辑均直接对接 NBA 官网原生 API 以及 pbpstats 网站的底层数据接口的抓包。我个人网站的底层数据基建代码均开源于此。

---

#使用须知

* **国内体感时区校准**：针对中美 12-15 小时的时差，脚本内部已将美国东部时间（ET）的比赛日期自动 `+1d` 转换为国内球迷的体感更新日期，确保数据看板的呈现不落后。
* **低门槛与高通用性数据格式**：为了方便国内广大球迷和初级数据分析爱好者直接查看，所有爬虫的最终输出数据均已统一转化为 Microsoft Excel (.xlsx) 格式，避免了传统 CSV 文件在 Windows 系统下常见的中文乱码问题。在实际网站的搭建过程中，我输出使用的则是parquet
* **高鲁棒性与智能增量架构**：针对常规赛、附加赛、季后赛设计了自动化双轨抓取逻辑。内置状态机与雷达探测机制，自动识别当前所处赛季阶段，并在季后赛期间执行增量更新，拒绝全量重复抓取和程序崩溃。逻辑就是先看看有没有季后赛数据，如果有说明当前正在季后赛当中，那就不会更新常规赛数据

## 脚本功能清单

目前仓库已上线的 4 大核心数据抓取代码：

1. `daily_player_log.py`
   * **功能**：获取 NBA 官网球员单赛季所有比赛的基础数据，包括比赛日期比赛对手，也可以加权计算真实命中率等高阶数据。
   * **亮点**：已包含国内时区转换逻辑，作为其他依赖单场比赛 ID 脚本的“雷达基础表”，比如季后赛球员对位数据，如果需要爬这部分代码的话务必要先抓取这个每日数据。

2. `team_stats.py`
   * **功能**：抓取 pbpstats 网站的球队级（本队与对手）常规赛及季后赛高阶统计数据。
   * **亮点**：防报错类型转换设计，并严格限制了请求频率以保护本地 IP。

3. `team_playtype.py`
   * **功能**：调用 NBA 官方 Synergy 接口，获取球队各项进攻与防守方式（如面框单打、挡拆、转换进攻、空切等）的频率与效率指标。
   * **亮点**：自带智能增量更新逻辑，季后赛开打后自动冻结并保留常规赛历史数据。

4. `player_matchup.py`
   * **功能**：获取球员常规赛/季后赛的对位数据，常规赛基于球队id获取，季后赛基于单场比赛拼凑合并
   * **亮点**：针对官方季后赛接口经常宕机的问题，内置了基于 Game ID 的精准降维单场嗅探器，确保任何阶段都能拿到最新对位数据。
